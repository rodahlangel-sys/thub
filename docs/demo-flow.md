# 平台演示流程

本文档用于本地演示武汉高校家教平台的主要使用路径。演示前请确认已经启动项目并完成数据初始化。

## 准备工作

```bash
npm install
npx prisma generate
npx prisma migrate deploy
npx prisma db seed
npm run dev
```

访问地址：

```text
http://localhost:3000
```

## 公开页面演示

1. 未登录访问 `/about`，查看关于平台。
2. 未登录访问 `/terms`，查看用户协议。
3. 未登录访问 `/privacy`，查看隐私政策。
4. 未登录访问 `/service-rules`，查看服务规则。
5. 未登录访问 `/refund-rules`，查看退款规则。
6. 未登录访问 `/safety`，查看安全提示。
7. 在首页底部查看页脚规则链接。

## 注册演示

1. 打开 `/register`。
2. 不勾选“我已阅读并同意《用户协议》和《隐私政策》”，提交后应看到中文错误提示。
3. 勾选协议后注册家长或大学生家教账号。
4. 注册成功后进入对应工作台。
5. 管理员进入用户详情页，可以看到用户协议同意时间和隐私政策同意时间。

## 家长演示路径

1. 打开首页，进入登录页。
2. 使用家长账号登录，进入家长工作台。
3. 进入“完善家长资料”，检查所在区域、详细地址和孩子情况。
4. 进入“发布家教需求”，填写年级、科目、区域、上课方式、预算和期望时间。
5. 进入“智能推荐”，选择一条需求并查看推荐老师。
6. 在推荐老师列表中查看匹配度、推荐理由和风险提醒。
7. 点击“预约这位老师”，填写预约时间、地点、课时和价格。
8. 创建订单后进入订单详情，状态为“待老师确认”。
9. 老师确认接单后，家长在订单详情点击“去支付”。
10. 支付前查看服务规则和退款规则入口。
11. 选择“模拟支付”，支付成功后订单进入“已担保”状态。
12. 老师提交课后反馈后，家长在订单详情查看反馈并确认服务完成。
13. 订单完成后进入评价页，提交四项评分和文字评价。

## 大学生家教演示路径

1. 使用大学生家教账号登录，进入家教工作台。
2. 进入“完善家教资料”，检查学校、专业、科目、区域、价格和可上课时间。
3. 进入“预约订单”，查看家长发来的预约。
4. 在订单详情中点击“确认接单”，订单进入“待家长支付”。
5. 家长支付后，订单进入“已担保”。
6. 老师点击“标记开始服务”，订单进入“服务中”。
7. 进入课后反馈页面，填写辅导内容、学生掌握情况、存在问题和下次建议。
8. 提交后订单进入“待家长确认”。
9. 家长确认并评价后，老师可在“收到的评价”中查看评价记录。

## 管理员演示路径

1. 使用管理员账号登录，进入管理员后台。
2. 在运营看板查看用户、家教、需求、订单、支付、退款、评价和通知统计。
3. 进入“用户管理”，按角色、账号状态或关键词筛选用户。
4. 打开用户详情，查看家长或大学生家教的资料与相关统计。
5. 查看用户协议同意时间和隐私政策同意时间。
6. 对普通用户执行禁用或恢复操作，管理员不能禁用自己的账号。
7. 进入“家教审核”，查看待审核家教并处理认证状态。
8. 进入“需求管理”，查看家长发布的需求。
9. 进入“订单管理”，查看订单双方信息、金额、状态和支付记录。
10. 进入“支付记录”，查看模拟支付流水。
11. 进入“退款管理”，查看退款申请，执行同意或拒绝审核。
12. 进入“评价管理”和“课后反馈”，查看服务完成后的记录。

## 退款演示路径

1. 家长选择一个已支付但未完成的订单。
2. 进入订单详情，点击“申请退款”。
3. 退款申请前查看退款规则入口。
4. 填写退款原因、说明和申请金额。
5. 提交后订单进入“退款申请中”。
6. 管理员进入“退款管理”，打开退款详情。
7. 同意退款后，订单状态变为“已退款”，支付状态变为“已退款”，退款状态变为“已通过”。
8. 拒绝退款时需要填写原因，家长订单详情会展示拒绝原因。

## 权限检查

- 未登录访问家长、大学生或管理员工作台，会跳转到登录页。
- 家长不能访问大学生工作台和管理员后台。
- 大学生家教不能访问家长工作台和管理员后台。
- 管理员不能进入普通用户工作台，会回到管理员后台。
- 非订单所属家长不能查看、支付、退款或评价别人的订单。
- 非订单所属老师不能处理别人的预约或提交反馈。
- 已禁用账号不能登录，已登录后被禁用也不能继续访问受保护页面。

## 注意事项

- 当前支付为模拟担保支付，不会产生真实扣款。
- 支付与退款已保留提供方接口结构，后续可以接入支付宝或微信支付。
- 本地演示中如需要恢复初始数据，可重新执行 `npx prisma db seed`。
