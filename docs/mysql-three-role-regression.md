# THub MySQL 三角色登录与受控写入回归

验证日期：2026-06-21

## 1. 测试范围与方式

本轮仅在 CloudBase MySQL 测试数据库 `thub_test` 中创建带唯一前缀的临时数据。测试通过本地 MySQL 模式 Next.js 服务提交真实页面表单和 Server Action；浏览器自动化运行环境不可用，因此未声称完成可视化浏览器操作。数据库直连仅用于准备临时管理员、读取断言、验证数据库约束和精确清理本轮夹具。

测试未修改 `.env`、默认 `DATABASE_URL`、SQLite schema 或业务代码，未运行 seed，也未部署应用。测试密码只保存在被 Git 忽略的临时夹具文件中，并已随清理删除。

- 临时家长和临时家教通过真实注册 Action 创建，临时管理员由显式 `--apply` 的准备脚本按现有 bcrypt 算法创建。

- 临时禁用家长后，旧登录 Cookie 被拒绝并跳转登录页；恢复状态后重新登录成功。
- 家长无法读取家教私有证明材料，家教本人和管理员可以读取。
- 家教上传测试图片并提交审核成功；审核前不满足正常推荐条件，管理员审核通过后状态正常。
- 重复审核请求被业务规则拒绝，未产生重复审核通知。
- MySQL 生成列唯一约束拒绝同一家教同时存在第二份有效在校证明。

## 4. 订单与退款流程

- 家长资料、家教资料和两条临时需求均通过真实 Action 创建。
- 同一需求和家教的重复预约被拒绝，未产生重复订单。
- 两条订单均完成家教确认和家长模拟支付，金额由服务端现有规则确定。

## 7. 可复用命令

- `npm run dev:mysql`：使用被忽略的 CloudBase 外网连接启动 MySQL 模式。
- `npm run mysql:e2e:prepare`：默认 dry-run；追加 `-- --apply` 才创建临时管理员和夹具。
- `npm run mysql:e2e:run`：通过真实 HTTP 页面表单和 Server Action 执行三角色回归。
- `npm run mysql:e2e:cleanup`：默认 dry-run；追加 `-- --apply` 才精确清理当前夹具。
- `npm run check:mysql-migration-result`：只读核对 MySQL 与 SQLite 基线。

## 8. 正式切换前事项

MySQL 数据、结构、本地只读运行和三角色写入流程已经验证。正式切换前仍需在 CloudBase Run 真实运行环境验证内网连接、生产变量注入、连接池行为，以及持久私有证明材料存储挂载和权限；还需要准备维护窗口、最终增量同步和明确回滚方案。本轮不代表这些运行环境事项已经完成。
