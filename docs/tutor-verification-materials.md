# 大学生家教证明材料功能说明

## 当前功能

大学生家教在“我的资料”页面可以上传证明材料，材料会和文字资料一起提交给平台管理员审核。证明材料只用于平台核验，不会向家长、其他大学生或未登录用户公开。

本功能不包含头像上传，也不会把学生证、校园卡或证明图片用于公开头像、老师卡片或推荐卡片展示。

## 大学生提交流程

1. 大学生登录后进入“我的资料”。
2. 填写学校、专业、可辅导科目、服务区域、价格区间等文字资料。
3. 上传“学生证或校园卡”，也可以上传学校正式出具的在读证明图片。
4. 选填其他能力证明，最多 5 张。
5. 确认资料和材料后点击“保存并提交审核”。
6. 提交后资料进入审核中状态，审核期间不能替换或删除本次材料。

## 管理员审核流程

1. 管理员进入“家教认证审核”。
2. 打开某位大学生的审核详情。
3. 查看文字资料、在校证明和其他能力证明。
4. 点击缩略图可打开原图。
5. 通过审核后，家教资料和本次提交材料都进入已通过状态。
6. 驳回审核时必须填写原因，大学生会在“我的资料”中看到原因。

## 材料类型

必填材料：

- 学生证或校园卡
- 学校正式出具的在读证明图片

选填材料：

- 教师资格相关证明
- 英语等级证明
- 学科竞赛证书
- 奖学金或荣誉证明
- 专业能力证书
- 家教经历证明
- 其他与辅导能力相关的证明

## 文件格式限制

仅支持：

- JPG / JPEG
- PNG
- WebP

不支持：

- SVG
- GIF
- PDF
- 可执行文件
- 文件内容与扩展名不一致的伪装图片

服务端会同时校验 MIME Type、文件扩展名和图片文件头。

## 文件大小限制

单张图片最大 5MB。

其他能力证明最多上传 5 张。

## 数据模型

新增模型：

- TutorVerificationDocument

新增枚举：

- TutorDocumentType
- TutorDocumentStatus

数据库只保存文件元信息和私有存储标识，不保存图片二进制内容，也不保存 Base64。

## 文件保存目录

本地私有文件保存到：

```text
.data/tutor-verification/
```

`.data/` 已加入 `.gitignore`，证明图片不会提交到代码仓库。

数据库中只保存不可预测的 `storageKey`，不保存服务器绝对路径。

## 私有访问规则

证明材料通过受保护接口读取：

```text
/api/tutor-documents/[documentId]
```

访问规则：

- 大学生本人只能查看自己的证明材料。
- 管理员可以查看所有审核材料。
- 家长不能查看任何证明材料原图。
- 其他大学生不能查看他人的证明材料。
- 未登录用户不能查看任何证明材料。

接口返回：

```text
Cache-Control: private, no-store
```

无权限时返回统一的 404，不暴露材料是否存在。

## 家长可见范围

家长端只显示大学生是否通过平台认证。

家长看不到：

- 学生证原图
- 校园卡原图
- 在读证明原图
- 能力证明原图
- 文件名
- 存储标识
- 服务器路径

## 驳回和重新提交

审核被驳回后：

1. TutorProfile 进入未通过状态。
2. 本次提交材料进入需修改状态。
3. 驳回原因会显示在大学生资料页。
4. 大学生可以替换在校证明。
5. 大学生可以删除或替换其他能力证明。
6. 重新提交后再次进入审核中状态。

## 文件替换与删除规则

- 上传新的在校证明后，会替换旧的当前在校证明。
- 新文件保存和数据库记录创建成功后，才删除旧文件。
- 数据库写入失败时，会清理已经保存的新文件。
- 审核中的材料不能删除或替换。
- 已驳回或待提交的材料可以删除或替换。
- 文件丢失时页面不会崩溃，会显示普通加载失败状态。

## 测试方式

建议测试：

1. 上传 JPG、PNG、WebP 图片。
2. 上传超过 5MB 的图片，应被拒绝。
3. 上传 SVG、GIF、PDF，应被拒绝。
4. 上传伪装成图片的非图片文件，应被拒绝。
5. 不上传在校证明时提交审核，应被拒绝。
6. 上传超过 5 张其他能力证明，应被拒绝。
7. 审核中尝试替换材料，应被拒绝。
8. 管理员查看缩略图和原图。
9. 管理员不填写原因驳回，应被拒绝。
10. 家长访问材料接口，应无法查看。
11. 另一个大学生访问材料接口，应无法查看。

## 已知限制

- 当前使用本地私有文件目录，适合本地开发和单机部署。
- 如果后续部署到无持久文件系统的云平台，需要切换为对象存储，并保持相同的私有鉴权接口。
- 当前不做图片压缩、OCR、自动打码或证件内容识别。
- 当前不提供头像上传能力。
