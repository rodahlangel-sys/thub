# 项目规则

## 基本定位

- 第一版必须是真实可运行平台，不是空壳模板。
- 平台面向武汉市区家庭与高校大学生，围绕大学生家教信息交互逐步建设。
- 每次开发必须保证 `npm run dev` 可以运行。
- 不允许随意更换技术栈；当前技术栈为 Next.js、TypeScript、App Router、Tailwind CSS、Prisma 和 SQLite。

## 数据规则

- 业务数据必须保存到 SQLite。
- 不使用 `localStorage` 作为主要数据存储。
- 所有服务端数据库访问统一通过 Prisma Client。
- 未接入真实支付前，先实现支付状态、担保状态和退款流程，不模拟真实扣款。

## 页面与内容

- 所有页面文字使用中文。
- 首页、业务页面、后台页面都应使用项目自己的样式，不保留默认 Next.js 示例风格。
- 新功能必须服务于真实平台流程，不创建无法运行或无法保存数据的静态假页面。

## 阶段边界

- 本阶段只完成项目基础底座。
- 本阶段不实现注册登录功能。
- 本阶段不实现业务页面。
- 后续阶段按角色系统、认证、需求发布、家教入驻、匹配、订单、支付状态、退款、反馈评价和管理员数据看板逐步推进。
